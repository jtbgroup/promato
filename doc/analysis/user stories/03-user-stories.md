# Promato — User Stories

## Authentication

**US-001** — As a user, I want to log in with my username and password so that I can access the application.  
**US-002** — As a user, I want to log out so that my session is closed securely.  
**US-003** — As a user, I want to change my password so that I can maintain account security.  
**US-004** — As an admin, I want to create user accounts so that new team members can access the app.  
**US-005** — As an admin, I want to assign roles to users so that permissions are correctly managed.  
**US-006** — As an admin, I want to deactivate a user account so that former members lose access.

## Projects

**US-010** — As a project manager, I want to create a project with name, code, description, dates and budget so that it is registered in the system.  
**US-011** — As a project manager, I want to invite users to my project and assign them a role so that collaboration is structured.  
**US-012** — As a project manager, I want to archive a completed project so that it remains visible but read-only.  
**US-013** — As any project member, I want to view the project dashboard so that I have a global overview of progress and consumption.

## PBS

**US-020** — As a project manager, I want to add a root PBS node to a project so that I can start decomposing the work.  
**US-021** — As a project manager, I want to add child nodes to any PBS node so that I can decompose work recursively.  
**US-022** — As a project manager, I want to edit a PBS node (title, description, effort, status, assignees) so that information stays up to date.  
**US-023** — As a project manager, I want to delete a PBS node (and its subtree) so that obsolete work packages are removed.  
**US-024** — As a member, I want to update the status of a PBS node I'm assigned to so that progress is tracked.  
**US-025** — As any user, I want to view the PBS as a tree and as a flat list so that I can navigate the structure easily.  
**US-026** — As a project manager, I want to reorder PBS nodes via drag-and-drop so that the structure reflects priority.

## Time Tracking

**US-030** — As a member, I want to log hours spent on a PBS node so that my contributions are recorded.  
**US-031** — As a member, I want to view my weekly timesheet so that I can review and correct my entries.  
**US-032** — As a project manager, I want to see time logged per PBS node so that I can monitor consumption.  
**US-033** — As a project manager, I want to see planned vs actual effort per node so that I can identify deviations.

## User Journey

**US-040** — As a project manager, I want to create a user journey scenario with steps so that user flows are documented.  
**US-041** — As a project manager, I want to link journey steps to PBS nodes so that functional coverage is traceable.  
**US-042** — As a project manager, I want to display user journey progress (% complete per step) so that stakeholders can see advancement.  
**US-043** — As a project manager, I want to export a user journey to PDF so that it can be shared externally.

## HLR

**US-050** — As a project manager, I want to generate a HLR document from my PBS and user journeys so that requirements are formalized.  
**US-051** — As a project manager, I want to edit non-functional requirements in the HLR so that the document is complete.  
**US-052** — As a project manager, I want to export the HLR to PDF so that it can be submitted to stakeholders.
